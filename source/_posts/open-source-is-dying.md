最荒唐的信号，来自 vLLM。

一个学生在辅导机构指导下，向 vLLM 提交了一个 PR。问题在于，这个 PR 针对的是一个并不存在的问题。更夸张的是，提交之后，他还在群聊里问老师「接下来怎么处理」，随后又通过大量 @ 维护者来争取关注。最后，这个 PR 成了社交平台上的宣传素材：恭喜同学 PR 合入 vLLM 社区，见证并参与 AI infra 社区发展。

这几句话听起来像喜报，其实是开源社区最不该承受的东西。

培训机构需要案例，学员需要简历，维护者需要付出时间。最后真正承担筛选成本的，是开源社区。

这件事最让人不适的地方，并不只是「学生水平不够」。水平不够可以学，可以从文档改起，可以从小问题做起，可以先问清楚再动手。真正让人愤怒的是，有人把开源项目当成简历镀金场，把维护者的善意当成免费背书，把社区里的真实协作包装成「学员成功参与顶级开源项目」。

![被做成流水线的简历 PR](/gallery/open-source-is-dying/resume-factory.jpg)

真正做过开源的人都知道，一个 PR 不是点一下提交按钮就算贡献。维护者要判断问题是否真实存在，要看方案是否合理，要读代码，要跑测试，要讨论兼容性，还要考虑这段改动以后由谁维护。每一个被打开的 PR，都会在维护者那里生成一笔债。

这笔债，本来应该花在真实 bug、真实用户、真实需求上。现在，它被拿去给培训班产出「成功案例」。

这比普通垃圾 PR 更伤人。普通垃圾 PR 也许只是能力不足，但这种 PR 背后有一套明确的利益链：机构收钱，学生拿经历，项目拿噪音，维护者拿压力。开源社区成了被消耗的一端，而且几乎没有拒绝的余地。因为一旦维护者语气重一点，就可能被说成不友好；一旦维护者关闭 PR，就可能被说成不欢迎新人；一旦维护者沉默，又会被更多低质量提交淹没。

所以，Open Source 社区正在死去。它不是死在没有 star，也不是死在没人 fork。它先死在维护者开始怀疑每一个新来的贡献者：你是真的想让项目变好，还是只想把我的项目写进你的简历？

过去，开源贡献是工程能力和社区信用的积累。你读代码，提问题，修 bug，接受 review，在来回沟通里慢慢获得信任。今天，有人想把这套过程压缩成简历流水线：找到热门项目，制造一个 PR，合进去，截图，发宣传，写进履历。

这会把开源社区最重要的东西毁掉。

![消失的信任](/gallery/open-source-is-dying/trust-fading.jpg)

开源真正珍贵的地方，是陌生人之间仍然愿意互相信任。维护者愿意相信你提交 PR 是为了解决问题，贡献者愿意相信维护者的 review 是为了让项目更好。可当越来越多 PR 背后站着培训机构、AI 代理、简历包装和流量宣传，这种信任就会一点点消失。

以后大项目只会越来越防御。新人 PR 更难被信任，维护者更少回应，Issue 模板更长，自动关闭规则更多。真正想参与开源的人，也会被这些简历流水线拖累。他们会发现，自己还没开始贡献，就已经先背上了别人制造出来的坏印象。

vLLM 这次差点中招，暴露的不是某一个学生的问题，而是开源生态正在被一种新的低质量输入围攻。AI 生成的 Issue 和 PR 是一种，培训机构批量制造的简历 PR 是另一种。它们看起来不同，底层逻辑很像：提交者用极低成本制造「贡献」的外观，把高成本的判断、验证、解释和善后全部推给维护者。

而在这场不对称的消耗战里，最先倒下的，一定是那些曾经毫无保留敞开大门的人。
